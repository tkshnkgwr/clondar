# Changelog

## [Unreleased]
### Added
- **コード全体へのドキュメントコメント（RustDoc/JSDoc）追加**:
  - Rust 側 (`lib.rs` / `main.rs`) に RustDoc コメントを追加。
  - フロントエンド React 側 (`main.jsx` / `App.jsx` / 各コンポーネント / ユーティリティ) に JSDoc コメントを追加し、引数 (`@param`) や戻り値 (`@returns`) のドキュメンテーションを整備。
- **ガイドラインへのドキュメンテーション同期ルールの追加**:
  - `.agents/AGENTS.md` に「9. ドキュメントコメント（RustDoc/JSDoc）の作成と同期」のセクションを追加。今後の仕様変更時にドキュメントコメントを同期して更新すること、またコメントの日本語表記を原則とすることをルール化。

### Changed
- **エラーメッセージおよびログ出力等の日本語化**:
  - `lib.rs` や `main.rs` 内の `expect` エラーメッセージ、`tauri.js` 内のデバッグログメッセージなどを日本語に翻訳・統一。

### Fixed
- **Viteビルド時の動的インポート警告の解消**:
  - `HolidaysManager.jsx` と `holidays.js` において、`@tauri-apps/api/core` や `reloadHolidays` が一部の関数内で動的インポート (`await import(...)`) されていたため、Viteビルド時にチャンク分割できないという警告が発生していた問題を、ファイル上部での静的インポート (`import`) に統一することで解消。

### Changed
- **バージョン管理の一元化とバンプスクリプトの修正**:
  - `src-tauri/tauri.conf.json` をアプリバージョンの「唯一の正のソース (SSOT)」とし、フロントエンド（`App.jsx`）でのバージョン表記は起動時に Tauri API を通じて動的に取得するように変更。
  - これに伴い、バージョンアップ時に書き換えるファイルを `tauri.conf.json` と `Cargo.toml` の2ファイルに限定。
  - 自動バージョンアップスクリプト (`scripts/bump-version.ps1`) を、上記2ファイルのみを確実に更新するシンプルな処理にリファクタリングし、置換エラーなどのバグを防止。
- **README ドキュメントのプレースホルダー画像をローカルスクリーンショットに置き換え**: ダミー画像のプレースホルダーを、ブラウザ上で実際に動作させたアプリ UI のキャプチャ画像 (`docs/assets/screenshot.png`) に置き換え、ドキュメントのビジュアルを最適化。
- **ソースコード内コメントの日本語化**:
  - `src-tauri/src/main.rs`、`ui/src/utils/holidays.js`、`ui/src/utils/tauri.js` に存在していた一部の英語コメントを、分かりやすい日本語コメントに翻訳・修正。
- **手動リリース運用への移行と自動バンプの廃止**:
  - `main` ブランチへのプッシュ時に自動でバージョンを上げてコミット・タグプッシュを行っていた `bump-version.yml` ワークフローを削除。
  - 今後はローカルで `scripts/bump-version.ps1` を実行してバージョンを更新し、Git タグ（`v*`）を作成・プッシュした時のみ自動リリースビルド（`release.yml`）が走るよう運用を変更（SnippetFlowと同様の方式）。

## [1.3.1] - 2026-07-06
### Added
- **自動リリース・バージョンバンプワークフローの追加**: `main` ブランチへのプッシュ時に自動的にパッチバージョンを上げてタグ付け・プッシュを行う `bump-version.yml` ワークフローを導入。
- **リリースビルド最適化設定の追加**: `src-tauri/Cargo.toml` に `[profile.release]` の最適化設定（LTO、シンボルストリップ等）を追加し、ビルドバイナリの軽量化を実現。

### Changed
- **共有ライブラリ `common_lib` 参照設定の変更**: `src-tauri/Cargo.toml` 内の `common_lib` の参照を Git 依存からローカルパス依存に戻し、CI/CD (`ci.yml`, `release.yml`) 側で `common_lib` を並行してチェックアウトしてビルドする方式へ移行。これに伴い、ローカルオーバーライド用の一時ファイル `src-tauri/.cargo/config.toml` を削除。

### Fixed
- **ダークモード時の祝日設定画面の配色修正**:
  - 定義数カード（「の日」等の統計ボックス）および祝日一覧のタイトルヘッダーで、Tailwindの無効なカラークラス `slate-850` が指定されていた箇所を `slate-800` に修正。これにより、ダークモード適用時にこれらの要素が白色（または明るい色）のままになる問題を解消。
  - 祝日リストアイテムのホバー背景クラスを `dark:hover:bg-slate-850/50` から `dark:hover:bg-slate-800/50` へ修正。
- **CI ワークフローのコンパイルエラーの修正**:
  - `ci.yml` において、Tauri のコンパイル時マクロ（`tauri::generate_context!`）が `frontendDist` (`ui/dist`) の存在を必要とするため、`cargo clippy` の実行前にフロントエンドのビルド（`npm run build`）を行うステップを追加。これにより、CI 上で発生していた `proc macro panicked` エラーを解消。

## [1.3.0] - 2026-07-03
### Added
- **祝日設定マネージャーへのビジュアル編集・保存機能の追加**: アプリ上の「祝日設定」画面で、直接固定祝日の「追加」や「削除」を行える編集フォームを追加。編集内容は Tauri コマンド経由で OS のユーザーローカルデータ領域（LocalAppData）へ安全に永続化保存され、カレンダー表示へ即時反映されるようにアップデート。
- **共有クレート `common_lib` の統合と「祝日設定マネージャー」機能の追加**: プロジェクト間共有クレートの `compute_diff` (LCS差分アルゴリズム) および `count_occurrences` を利用し、カレンダーアプリ内に外部祝日設定ファイルの整合性やカスタマイズ差分、統計情報を視覚的に確認・検証できる「祝日設定マネージャー」画面を追加。Tauri コマンド連携およびシリアライズ対応を含む。
- **GitHub Actions ビルドワークフローの更新**: 共有クレート `common_lib` の GitHub リポジトリ登録に伴い、ワークフロー内で自動チェックアウト・配置を行うステップを追加し、クラウド上でもコンパイルおよびリリースビルドが正常に通るように修正。
- **開発ガイドラインの更新**: プロジェクトルール定義ファイルである `.agents/AGENTS.md` に、`docs/DEVELOPMENT.md`（開発用ドキュメント）を自動更新の義務付け対象として明記。
- **システムトレイ（タスクトレイ）常駐機能の追加**: 表示/非表示、最前面切替、位置リセット、終了を右クリックから操作可能なシステムトレイメニューを Rust バックエンド側で実装。

### Fixed
- **画面全体のレイアウトとモーダルUIの最適化**:
  - ウィンドウ幅を `1100px` から `1180px`（WindowFrameは `1140px`）に拡張し、カレンダーヘッダーの折り返し不具合（年月の改行、ボタン名の2行化）を解消。ヘッダー要素に `whitespace-nowrap` および `flex-shrink-0` を追加。
  - 祝日設定マネージャー画面のサイズを高さ `460px`・幅最大 `5xl` に最適化し、外枠をはみ出して見切れる問題を解消。さらに左側サイドバーに `overflow-y-auto` スクロールを適用し、すべての統計データに正常にアクセス可能に修正。
- **ウィンドウクローズ時のパニック不具合の修正**: フロントエンドの `❌` ボタンや Esc キーでウィンドウを閉じた（破棄した）後にシステムトレイから再操作した際、Webview 喪失によって発生する `unwrap` パニック（`FailedToReceiveMessage`）を解消するため、ウィンドウクローズ要求を非表示（Hide）へフック・すり替える常駐化処理を導入。トレイイベント内の `unwrap` も安全なエラーハンドリングへ全面改修。
- **WebView2 終了時の警告エラーの解消**: システムトレイから終了する際、`app.exit(0)` による強制終了から、正規のウィンドウ破棄を伴うクローズフローへ移行しました。これに伴い通常のクローズ操作と明示的なアプリ終了を区別するための `QuittingState` 状態管理を導入し、WebView2 解除時の Win32 エラー警告（Error 1412）を完全に解消。
- **完全オフライン動作可能なローカルビルド環境の導入 (Vite + React)**: すべてのフロントエンド依存関係を Vite でローカルパッケージ化。CDN への接続を完全に排除し、PC が完全オフラインの状態でもウィジェットが正常動作する仕組みを構築。
- **外部祝日設定ファイル (`holidays.json`) の導入**: 日本の祝日定義を `ui/public/config/holidays.json` に外部化し、法改正時もコードを変更せずに JSON 設定 of 変更のみで対応できる仕組みを構築。
- **開発者向けドキュメント (`docs/DEVELOPMENT.md`) の新設**: Vite+React アーキテクチャ、モジュール分割、ビルド・起動手順、トレイ機能、外部祝日定義ルールを解説する技術ドキュメントを作成。
- **`README.md` / `README.ja.md` へのステータスバッジの追加**: ビルド状況、リリースバージョン、動作環境 (Rust 1.96.0, Node.js >=26.4.0, Tauri v2) を示すシールドバッジを上部に追加し、技術スタック表示の不整合を修正。
- **`.agents/AGENTS.md` 開発ガイドラインの更新**: ドキュメント自動更新ルールの中に、README 編集時にステータスバッジの保守管理を行う要件を明記。
- **インストーラーの日本語化対応 (Windows)**: NSIS インストーラーでの日本語・英語多言語対応（インストール時の言語選択画面表示機能付き）、および WiX (MSI) での日本語専用（ja-JP）と英語専用（en-US）パッケージ of 並行ビルド設定を導入。

### Changed
- **共有ライブラリの二重起動防止機能（Named Mutex）の組み込み (2026-07-10)**:
  - `common_lib::check_single_instance` の API 設計変更（Result を返す仕様）に合わせて、`clondar` 起動時に Named Mutex を用いた二重起動チェックを実装。
  - 既に別インスタンスが起動している場合は、エラーを出力して安全にプロセスを終了（`std::process::exit(0)`）する処理をメインプロセスの初期化フェーズへ統合。
- **GitHub Actions および Dependabot 連携の修正とCIの新規導入 (2026-07-06)**:
  - Dependabot で発生していた `common_lib` パス依存関係のエラー（`path_dependencies_not_reachable`）を解消するため、`Cargo.toml` の `common_lib` を Git 依存関係に変更。
  - ローカル開発での利便性を損なわないよう、`.cargo/config.toml` によるローカルパス（`../../common_lib`）へのオーバーライドを導入し、`.gitignore` に除外設定を追加。
  - リリースワークフロー（`release.yml`）内のアクションの非推奨バージョンを安定版（`checkout@v4`, `action-gh-release@v2`）に修正し、ジョブレベルの作業ディレクトリ指定（`clondar`）およびリリースアセットの出力先パス指定を正確に設定。
  - プッシュ・プルリクエスト時にコード品質を自動検証（`cargo fmt`, `cargo clippy`, `cargo test`）する CI ワークフロー（`ci.yml`）を新規導入。
- **フロントエンドのモジュール分割と JSX 移行**: UI コンポーネントを `Clock.jsx`, `Calendar.jsx`, `App.jsx`, `main.jsx` に機能別にクリーンに分割し、標準的な React/JSX 記法で記述。
- **ビルドコマンドの自動化**: `tauri.conf.json` での `beforeDevCommand`, `beforeBuildCommand` の設定を Vite 用に再適用し、Tauri 起動時に自動でコンパイルされるよう修正（Tauri CLI が `frontendDist` に基づき自動的に `ui/` ディレクトリを作業ディレクトリとして扱うため、標準的な `npm run dev` / `npm run build` のまま動作します）。
- **環境バージョンの最新整合**: Rust バージョン要件を `1.96.0` に、Tauri 依存関係の表記を実際のビルドバージョン（tauri `2.11.3`, tauri-build `2.6.3`）と整合。
- **カレンダー「今月」ボタンのアイコン化**: カレンダー表示の「今月」ボタンをテキストから「家（ホーム）」のアイコンボタンに変更し、UIデザインの統一性を向上。

### Fixed
- **`.gitignore` における `node_modules` 除外の修正**: 先頭スラッシュ（`/node_modules/`）が指定されていたためルート直下しか除外されず、`ui/node_modules/` が Git の追跡対象に入ってしまっていたバグを、スラッシュを削除して再帰的に除外されるよう修正。
- **VS Code 設定スキーマの警告修正およびバージョンバンプ漏れの解消**: `.vscode/settings.json` における Rust フォーマッタの警告（未ロード時に発生する `editor.defaultFormatter` の警告）を項目削除によって修正し、最新バージョン `1.3.0` へのバンプ漏れを解消。
- **CDN版 framer-motion (UMD) 読み込み時のエラー修正**: CDNから読み込まれる `framer-motion` (現 `Motion`) のグローバルオブジェクトの構造に対して、`motion.div` や `motion.AnimatePresence` を正しくバインドできずに画面全体やカレンダーがクラッシュ・非表示になっていたバグを修正。Proxy による動的フォールバックを構築。
- **htm テンプレート内の JSX スタイルコメントの削除**: htm テンプレートリテラル内に JSX 形式のコメント `{/* ... */}` が残っていたため、パースエラーが発生してカレンダーや設定コントローラーなどの後半要素がレンダリングされていなかったバグを修正。

## [1.2.0] - 2026-06-15
### Fixed
- **位置記憶バグの完全修正**: 起動時の自動中央配置 (`center: true`) と JS による位置復旧ステップのタイムラグ中、一時的な極小座標や初期中央座標を誤取得して `localStorage` を誤上書き破壊してしまうクリティカルな競合バグを完全に修正。
- **位置復帰ガードの実装**: 設定のロードと位置復旧プロセス中（起動から1秒間）、手動移動監視のリスナーと 5 秒ポーリングの `localStorage` 保存処理を一時遮断（`isRestoringRef` によるガード）する安全な状態制御を開発。
- **DPIズレの終息**: スケール倍率（125%や150%）に依存する論理座標（Logical）ではなく、Tauri が出力する物理絶対座標（`Physical`）を用いて位置 get/set を一貫させることで、DPI環境での起動時ズレを恒久対策。
- **セキュリティ特権の設定**: 座標情報の取得とプログラムからのウィンドウ座標再配置を実現するため、`capabilities/default.json` へ `allow-outer-position` / `allow-set-outer-position` / `allow-set-position` の権限を拡張追加。

## [1.1.0] - 2026-04-16
### Added
- **Tauri v2 移行**: アプリケーションを Tauri v2 ベースのデスクトップウィジェットへ刷新。
- **透過ウィンドウ**: 背景透過、枠なし（Borderless）、影なし（Shadowless）を実装。
- **権限管理**: `capabilities/default.json` を導入し、ドラッグや終了のセキュリティ権限を適切に設定。
- **デバッグ機能**: 開発時のファイル同期確認用マーカー（後に削除済み）。

### Fixed
- **影の完全除去**: Windows OS 標準の影が透過背景に残る問題を Rust 側の `set_shadow(false)` で解決。
- **ドラッグ操作の安定化**: 透明部分でマウスイベントが透過する問題を `pointer-events: auto` で解決。
- **終了ロジック**: `Esc` キーおよび `❌` ボタンによる終了が Tauri v2 API で確実に動作するよう修正。
- **レスポンシブの固定**: デスクトップウィジェットとしての安定性を優先し、モバイル向けレスポンシブクラスを整理。

### Changed
- **デザイン刷新**: `WindowFrame` から物理的な枠線と影を削除し、よりミニマルな外観へ。
- **フォント変更**: デジタル時計のフォントをより力強い `Impact` 系へ変更。
