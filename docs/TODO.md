## 2. 直近のタスク (In Progress / Todo)

- [ ] **`common_lib` 参照方法の整合性修正**
  - **現状の課題**: `DEVELOPMENT.md` や `AGENTS.md` の開発ガイドラインでは、CIビルドエラーを防ぐため `Cargo.toml` の `common_lib` 依存を Git 参照にし、ローカル開発時のみ `.cargo/config.toml` の `paths` でオーバーライドすることになっている。しかし、現在 `src-tauri/Cargo.toml` に相対パス `{ path = "../../common_lib" }` が直接記述されており、不整合が発生している。
  - **対応策**: `Cargo.toml` を Git 参照に戻し、ローカル開発環境での動作には `.cargo/config.toml` によるパス上書きを徹底させる。
- [x] **フロントエンド側での JSON 簡易バリデーションの実装**
  - **現状の課題**: 祝日データの保存時、Rustバックエンドの `serde_json` による検証のみに依存しており、万が一フロントエンド側で壊れた JSON を送信しようとするとバックエンドエラーが発生する。
  - **対応策**: フロントエンド側で保存ボタンを押した直後に `JSON.parse` などの簡易的な構文チェックを行い、事前エラーを表示する。
- [x] **テストコードの整備**
  - **現状の課題**: GitHub Actions で `cargo test` を走らせるワークフローが定義されているが、プロジェクト側の具体的なユニットテストや UI コンポーネントのテスト（Vitest等）が充実していない。
  - **対応策**: 主要な計算ロジック（祝日算出、座標判定など）に対するテストケースを追加する。

## 3. 今後の拡張提案 (Backlog)

- [ ] **アプリ動作中の UI 表示言語の動的切り替え**
  - カレンダーの曜日や祝日名、設定ダイアログ内の各種テキストを、アプリ内の設定から「日本語 ⇄ 英語」へ動的に切り替えられるようにする多言語ローカライズ機能。
- [ ] **テーマカラーと透過度の詳細カスタマイズ**
  - 背景透過の度合い（ブラーの強度）をスライダーで細かく調整でき、テーマカラー（現在はブルーベース固定）を数種類から選べるようにするパーソナライズ設定機能。
- [ ] **簡易スケジューラー（ToDo・予定管理）機能**
  - カレンダーの日付セルをクリックすることで、ユーザーがその日の簡単なスケジュールや ToDo を登録・保存し、カレンダー上にインジケーターとして表示できるウィジェット拡張。
- [ ] **祝日定義ファイルのオンライン自動更新**
  - 祝日の法改正に自動で追随するため、起動時にGitHubなどの特定サーバーから最新の `holidays.json` をバックグラウンドで自動取得・更新する仕組み。
- [ ] **スタートアップ自動起動設定の UI 統合**
  - PC起動時にウィジェットを自動で起動させる「スタートアップ登録」オプションを、設定メニューやトレイメニューの UI から直接 ON/OFF 切り替えできる機能（Tauri の autostart プラグイン等の導入）。
