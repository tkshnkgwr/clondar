# 貢献ガイドライン (CONTRIBUTING.md) - Clondar

[English](../en/CONTRIBUTING.md) | **日本語版**

`Clondar` プロジェクトへの貢献に興味を持っていただきありがとうございます！
本ドキュメントでは、バグ報告、機能提案、プルリクエスト提出時のガイドラインを説明します。

---

## 1. 開発方針と重要原則

開発を行う際は、以下の基本方針を遵守してください。

1. **ローカル完結型 / オフライン動作の保証**:
   - アプリケーションは完全なオフライン環境でも動作するように設計されています。CDN 等への外部参照を排除し、すべてのアセットをローカル（Viteでビルドされた成果物）で完結させてください。
2. **透過・枠なし・影なしのビジュアル維持**:
   - Windowsデスクトップウィジェットとしての「究極のミニマリズム」を損なわないよう、ウィンドウ境界、タイトルバー、およびOS標準のウィンドウ影が確実に除去されている状態を保ってください。
3. **多言語ドキュメントの同期**:
   - 仕様変更や機能追加を行う際は、`docs/ja/` および `docs/en/` の両方のドキュメントを更新してください。

---

## 2. 開発環境のセットアップ

ビルドや開発には **Node.js (v26.4.0 以上推奨)** と **Rust (1.96.0 以上推奨)** が必要です。また、共有ライブラリ `common_lib` を並行開発する場合はローカルパスのオーバーライドを行ってください。

1. **リポジトリのクローン**:
   ```bash
   git clone https://github.com/tkshnkgwr/clondar.git
   cd clondar
   ```
2. **依存関係のインストール**:
   ```bash
   npm --prefix ui install
   ```
3. **開発モードでの起動**:
   ```bash
   cargo tauri dev
   ```

## 3. コミットおよびプルリクエスト手順

### コミットメッセージの規約
コミットメッセージには Conventional Commits 形式を使用してください：

- `feat:` 新機能の追加
- `fix:` バグ修正
- `docs:` ドキュメントの変更
- `refactor:` リファクタリング
- `perf:` パフォーマンス改善
- `test:` テストの追加・修正
- `chore:` ビルドスクリプトや設定の変更

### プルリクエスト作成前のチェックリスト
プルリクエストを送信する前に、以下のコマンドを実行し全て合格することを確認してください：

- [ ] `cargo test` （バックエンドユニットテスト合格）
- [ ] `npm --prefix ui run test` （フロントエンドテスト合格）
- [ ] `cargo clippy --all-targets -- -D warnings` （静的解析の警告ゼロ）
- [ ] `cargo fmt --check` （コードフォーマット準拠）
- [ ] `cargo doc --no-deps --document-private-items` （Rustdocビルドエラーゼロ）
