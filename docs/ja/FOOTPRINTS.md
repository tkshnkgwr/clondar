# フットプリント記録 (FOOTPRINTS.md) - Clondar

[English](../en/FOOTPRINTS.md) | **日本語版**

本文書は、`Clondar Pro` のリリースビルド時におけるバイナリサイズ、リソース消費量（メモリフットプリント）、およびビルドパフォーマンスの記録です。

---

## 1. 計測環境

- **OS**: Windows 11 64-bit
- **Toolchain**: Rust stable (`x86_64-pc-windows-msvc`), Node.js v26.4.0
- **ターゲットアセット**:
  - アプリケーション実行ファイル: `src-tauri/target/release/clondar.exe`
  - 配布用インストーラー: `src-tauri/target/release/bundle/nsis/*.exe` / `wix/*.msi`

---

## 2. フットプリント記録

### 2.1 バイナリおよびパッケージサイズ

| ビルドタイプ / アセット名 | サイズ | 備考 |
| :--- | :---: | :--- |
| `clondar.exe` (Debug) | 約 45 - 60 MB | デバッグシンボル・ログモジュール含む |
| `clondar.exe` (Release) | 約 10 - 15 MB | LTOおよびシンボルストリップ適用済 |
| NSIS インストーラー (`.exe`) | 約 5.0 - 7.5 MB | 圧縮済シングルインストーラー |
| WiX インストーラー (`.msi`) | 約 4.5 - 6.5 MB | 各ロケール専用の MSI パッケージ |

> **特徴**: WebView2 ランタイムをOS共有コンポーネントとして利用するため、Electron等（バイナリ単体で約 100MB 以上）と比較して、インストーラー容量および展開後のファイルサイズともに圧倒的に軽量なフットプリントを実現しています。

### 2.2 メモリフットプリント (RAM)

- **メインプロセス (Rust)**: 約 3.0 MB ~ 5.0 MB
- **レンダラープロセス (WebView2 / Edge)**: 約 25 MB ~ 45 MB
- **実行時ピーク総メモリ (合計値)**: 約 30 MB ~ 55 MB

> **特徴**: フロントエンド側でのアセットのローカル読み込みの最適化、定期ポーリングのインターバル制御、および不要なDOMレンダリングの抑制により、常駐型の時計・カレンダーウィジェットとしてシステムに負荷を与えないメモリ消費量を維持しています。

### 2.3 コンパイル性能 (ローカル環境目安)

- **クリーンビルド時間 (`cargo tauri build`)**: 約 60秒 ~ 120秒
- **増分ビルド時間 (`cargo tauri dev`ホットリロード)**: 約 1.0秒 ~ 3.5秒

---

## 3. リリースビルド最適化設定

バイナリサイズ削減とメモリ消費量抑制のため、`src-tauri/Cargo.toml` において以下のプロファイル最適化設定を適用しています：

```toml
[profile.release]
opt-level = 3        # パフォーマンス最適化
lto = true           # Link-Time Optimization の有効化
codegen-units = 1    # 単一コード生成による最適化向上
panic = "abort"      # パニックスタック展開の省略によるサイズ削減
strip = true         # シンボルおよびデバッグ情報のストリップ
```
