# セキュリティポリシー (SECURITY.md) - Clondar

[English](../en/SECURITY.md) | **日本語版**

本文書は、`Clondar Pro` プロジェクトにおけるセキュリティ方針および脆弱性報告手順をまとめたドキュメントです。

---

## 1. セキュリティ設計と安心性

`Clondar Pro` は以下の設計原則により、高いセキュリティと信頼性を保証しています。

1. **Tauri v2 によるサンドボックス & 権限分離 (Capabilities)**:
   - レンダラー（Webview）からRustバックエンド（OS機能）へのアクセスは、Tauriの厳格なセキュリティ機能により完全にサンドボックス化されています。
   - `capabilities/default.json` において、ウィンドウのドラッグ、終了、ピン留め、座標取得など、アプリケーションの動作に必要な最小限の実行権限のみを明示的にホワイトリスト登録して運用しています。
2. **Rust によるメモリ安全性**:
   - バックエンドのメインプログラムは Rust 言語で記述されており、コンパイル段階でメモリ安全性上の脆弱性（バッファオーバーフロー、ヌルポインタ参照、競合状態など）が排除されています。
3. **完全ローカル完結動作（オフライン保証）**:
   - アプリケーションは外部サーバーとの通信を行わず、完全にPC上のローカル資源のみで動作します。祝日の定義ファイルなどもローカルの AppData フォルダ内に閉じて処理されるため、中間者攻撃や情報漏洩のリスクはありません。

| :--- | :---: |
| 最新リリース (v1.3.x) | ✅ サポート |
| 古いリリース (v1.2.x 以前) | ❌ 非サポート |

---

## 3. 脆弱性の報告方法

`Clondar Pro` に潜在的なセキュリティ脆弱性を発見された場合は、公開 Issue を作成するのではなく、直接リポジトリ管理者またはセキュリティ窓口へ電子メールにて詳細をご報告ください。

ご報告の際は以下の情報を含めていただけると迅速な対応が可能です：
1. 影響を受けるアプリケーションのバージョンおよび詳細な実行環境（OSバージョンなど）
2. 脆弱性の詳細説明と再現手順（PoCコードや具体的な操作手順）

報告受領後、速やかに内容を検証し、修正プログラムの開発およびアップデート版のリリースを行います。
